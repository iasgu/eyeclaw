# Show Once 操作手册

## 1. 系统作用

Show Once 用来把浏览器里的真实操作流程转成：

- 可阅读的 SOP
- 可回放的步骤计划
- 监听器引导的关键帧分析结果

当前推荐使用方式是：

1. 先在 Edge 里开启监听
2. 真实操作一遍目标流程
3. 再让系统分析监听会话
4. 如有需要，再结合视频做验证

## 2. 使用前准备

### 2.1 启动前端服务

在项目根目录运行：

```powershell
.venv\Scripts\python.exe -m uvicorn app_web:app --host 127.0.0.1 --port 8018
```

打开：

`http://127.0.0.1:8018`

### 2.2 加载监听器扩展

如果手工加载失败，建议直接使用已经验证过的方式：

1. 打开新 Edge 窗口
2. 保持这个窗口不要关闭
3. 在扩展页确认能看到 `Show Once Listener`

### 2.3 启动监听

点开扩展弹窗，确认：

- `监听器接口地址` 是 `http://127.0.0.1:8018`
- 当前状态显示为 `未监听`
- 点击 `开始监听`
- 点击后状态会变成 `监听中`
- 系统会自动创建一个新的会话 ID

## 3. 主界面使用

### 3.1 最推荐的流程

1. 启动前端服务
2. 打开扩展弹窗
3. 点击 `开始监听`
4. 在 Edge 里按真实流程操作页面
5. 回到控制台，点击 `刷新监听器`
6. 确认右侧出现事件、截图数量和会话 ID
7. 点击 `分析监听会话`
8. 查看右侧 `回放步骤`

### 3.2 视频分析流程

视频分析现在必须参照监听器信息。

也就是说：

- 先要有监听会话
- 再去分析 `website.mp4` 或你上传的视频
- 如果没有监听会话，视频分析会报错

步骤：

1. 先完成一次真实监听会话
2. 保持监听会话存在
3. 上传视频或直接使用 `website.mp4`
4. 填写任务说明
5. 设置开始秒数、结束秒数、关键帧上限
6. 点击 `分析视频流程`

## 4. 扩展按钮说明

### 开始监听

- 开启采集
- 自动生成一个新的会话 ID
- 后续事件都会记到这个新会话里

### 停止监听

- 立即停止采集
- 不再接收新的事件

### 保存接口设置

- 保存监听器接口地址
- 保存客户端名称

### 重置会话 ID

- 手动换一个新的会话 ID
- 适合你想手工切换一次录制会话的时候使用

## 5. 常见问题

### 5.1 为什么我操作了页面，但控制台没有事件

检查这几项：

- 扩展是否真的加载成功
- 扩展弹窗里是否点击了 `开始监听`
- 状态是不是 `监听中`
- 前端服务是否还在运行
- 接口地址是否还是 `http://127.0.0.1:8018`

### 5.2 为什么视频分析失败

通常是以下原因之一：

- 当前没有监听会话
- 监听会话里没有关键事件
- 模型接口网络不可用
- `.env` 配置异常

### 5.3 为什么我点击开始监听后还想重新录一遍

最简单的做法：

1. 点击 `停止监听`
2. 再点一次 `开始监听`

这样系统会自动开始一个新的会话。
